# System Snapshot – מצב נוכחי (Version 1)

זהו צילום מצב קצר של המערכת שלי אחרי פתיחת הריפו החדש `ai-os`.

## ריפואים ומבנה

### `ai-os`  
- משמש כ"מוח המרכזי" (Single Source of Truth) של מערכת ה-AI שלי.
- מכיל:
  - `README.md` – תיאור כללי של הריפו.
  - `docs/REPO_AUDIT_make-ops-clean.md` – אודיט מפורט של הריפו הישן.
  - קבצי תיעוד נוספים שיתווספו בהמשך (חוקה, מיפוי, תיעוד סוכנים וכו').

### `make-ops-clean` (ריפו ישן)  
- מתפקד כמערכת הפעלה קיימת/ישנה של סוכני AI, MCP, אוטומציות, אבטחה ותיעוד.
- כרגע נמצא במצב של "ארכיון חי": לא נמחק, אבל לא נחשב עוד למקור האמת הראשי.
- מפורט באודיט:
  - `docs/REPO_AUDIT_make-ops-clean.md`

## כלים ומודלים (בגבוה)

### Claude Desktop:
- מחובר ל-GitHub (MCP).
- משמש לעבודה על הריפו `ai-os` ועל קבצים מקומיים.

### GPT (צ'ט + Agent Mode):
- משמש למיפוי ומעקב אחרי תכולת הריפו הישן.
- מייצר מסמכי אודיט ותובנות שאנו שומרים בתוך `ai-os/docs/`.

### כלים ואינטגרציות נוספים קיימים (Make, Telegram, GPT API וכו'):
- כרגע מתועדים בעיקר בריפו הישן.
- בהמשך נחליט מה מהם נכנס כ"אזרחים רשמיים" ל-AI-OS (ב-`tools/` ו-`workflows/`).

## איפה אנחנו עכשיו בתהליך

- ✔ נפתח ריפו חדש `ai-os` עם מבנה בסיסי.
- ✔ יובא אודיט מלא של הריפו הישן (`make-ops-clean`) לתוך `docs/REPO_AUDIT_make-ops-clean.md`.
- ⏳ בשלב הבא:
  - נתחיל לבחור "זהב" מהריפו הישן, לפי האודיט,
  - ולהעביר אותו בצורה מסודרת לתיקיות המתאימות ב-`ai-os` (docs/agents/tools/workflows/policies),
  - בלי לגעת בינתיים בריפו הישן עצמו.
