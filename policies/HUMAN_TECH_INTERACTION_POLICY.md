## 🎯 מטרת המדיניות

מדיניות זו מגדירה **חוק ברזל** במערכת AI-OS:

> **אור לא עושה עבודה טכנית.**

כל מה שטכני = באג ארכיטקטורה שצריך לפתור דרך סוכנים / Workflows / כלים – **לא דרך אור**.

**למה זה חשוב?**

1. **Scalability**: מערכת שתלויה באור לעבודה טכנית לא תתרחב
2. **Sustainability**: אור לא יכול להיות bottleneck
3. **Automation**: המטרה היא אוטומציה, לא עבודה ידנית
4. **Architecture**: אם משהו דורש אור טכני - זה באג עיצוב

**עקרון המפתח**:
```
אם סוכן/Workflow מבקש מאור לעשות פעולה טכנית →
זה לא "פתרון" → זה באג ארכיטקטורה שצריך לתקן
```

```

```

**תנאים**:
- פעולה אחת פשוטה (לחיצה אחת)
- ברור לחלוטין מה קורה
- לא דורש ידע טכני

---

#### 2. **הדבקת טוקן חד-פעמי (מקרי קיצון בלבד!)**
```

```

**תנאים קשיחים**:
- אין שום דרך אחרת (Automation נכשל)
- פעם אחת בלבד (לא שוטף)
- מתועד כBAG שצריך לתקן
- יש תוכנית להפוך לאוטומציה

**דוגמה**:
```
אור מדביק GitHub PAT ב-Claude Desktop config פעם אחת
→ אבל צריך task: "אוטומציה לניהול GitHub tokens"
```

---

#### 3. **Emergency Override (רק בחירום אמיתי!)**
```
🚨 מותר בחירום: פעולה טכנית אחת לעצור נזק
```

**תנאים**:
- חירום אמיתי (data loss, security breach)
- לא ניתן לחכות
- מתועד מיד אחר כך
- נעשה post-mortem

**דוגמה**:
```
Secret חשוף בגיטהאב → אור עושה force-push מיידי
→ אבל צריך post-mortem: "למה זה קרה? איך למנוע?"
```

## 📜 Operational Rules for All Agents

### 10 חוקים קשיחים:

#### 1. **Zero Technical Burden on Or**
```
כל בקשה שמערבת עבודה טכנית אצל אור = הפרת מדיניות
```

**אין חריגים.**

---

#### 2. **Tool Failure = DESIGN Mode**
```
אם הכלי שבור → DESIGN, לא "אור, תעשה ידנית"
```

**אין workarounds דרך אור.**

---

#### 3. **Approval ≠ Execution**
```
אור מאשר, סוכן/כלי מבצע
```

**תמיד.**

---

#### 4. **Secrets = Automation Only**
```
סיקרטים מנוהלים רק דרך Secret Manager / Automation
```

**אסור לבקש מאור להדביק/לעתיק.**

---

#### 5. **No Workarounds**
```
אם אין capability → תעד ועבור ל-DESIGN
```

**אל תמציא פתרונות חלופיים שמערבים אור.**

---

#### 6. **Git = Automation**
```
כל פעולת git נעשית דרך Claude/MCP
```

**אסור לבקש מאור git commit/push/clone.**

---

#### 7. **File Ops = MCP**
```
יצירה/עריכה/מחיקה של קבצים = דרך Filesystem MCP
```

**אסור לבקש מאור ליצור קובץ.**

---

#### 8. **UI = Browser/Windows MCP**
```
אינטראקציות UI = דרך Browser MCP / Windows MCP
```

**אסור לבקש מאור לפתוח אתר ולעשות X.**

---

#### 9. **Emergency Stop = Immediate**
```
"עצור עכשיו" → עצירה מיידית, לא "רק אסיים..."
```

**עוצרים הכל מיד.**

---

#### 10. **Respect Or's Time**
```
אל תבזבז זמן של אור על דברים שהמערכת יכולה לעשות
```

**תפקידו: החלטות, לא ביצוע.**
